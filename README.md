## 리액트 로그인 세션 과제

**1. 쿠키** : 사용자의 컴퓨터에 저장하는 작은 기록 정보 파일
  - 데이터를 클라이언트 측에 저장함
  - 데이터 크기에 제한이 없음
  - 클라이언트가 페이지를 요청하면 웹서버가 쿠키를 생성. 생성한 쿠키에 정보를 담아서 클라이언트에게 돌려 주면 클라이언트가 가지고 있다가 서버에 요청과 함께 쿠키 전송. 
  - but, 쿠키 데이터는 브라우저에 저장되어 있어서 보안에 취약함 -> 데이터를 서버에 저장하는 방식으로 발전

**2. 세션** : 일정 시간 동안 브라우저로 들어오는 요구를 유지시키는 기술
  - 데이터를 서버에 저장해 두었다가 클라이언트가 세션id를 서버에게 보내주면 서버가 이를 확인한 후 데이터를 전송
  - 세션id 값을 쿠키를 사용하여 저장하고 전송함
  
**3. 세션 기반 로그인**
  : 사용자가 '로그인 중임'을 서버가 기억하고 있는 로직
  - 유저가 로그인을 성공하면 서버는 세션 저장소에서 사용자 정보를 조회, 없으면 세션 id를 유저에게 발급
  - 발급된 세션id는 유저의 브라우저 쿠키에 저장
  - 클라이언트가 세션id와 함께 서버에 요청을 보낼때마다 서버는 세션 저장소에서 조회 후 로그인 여부를 결정함
